Probe Types thermistor model

Pick the thermistor model for each probe for accurate readings. "Leave unchanged" keeps the current calibration.

Calibration Offsets degrees, +/-

Food Targets sets the high alarm

Choose a meat preset or type a temperature. The dashboard shows time-to-done for each. Enable the toggle to give the 4th (Ambient) probe a target too.

PID Tuning

Adds a live P / I / D output breakdown card to the dashboard. Useful for understanding the control loop and validating auto-tune. Off by default (adds a little serial traffic).

PID Auto-Tune finds tuning for your cooker

Auto-tune cycles the blower on/off around the target to learn your cooker's response, then writes new PID values. It runs the fire hard - supervise it, and expect 20-40 min. The grill returns to automatic when done.

Fan & Servo

Lid Detection

When the pit drops this % below setpoint (lid opened), the fan pauses for this long. Auto-detected.

LCD & LEDs front panel

Backlight 0-255. Home screen layout and which event each LED reacts to (checkbox inverts). LED 0 is the wireless LED; 1-3 are on the LCD board.

Alarms blank = unchanged, −1 = off

Manual Fan Mode

To leave manual mode, set a temperature setpoint on the main screen.

Home Assistant (MQTT)

HeaterMeter auto-appears in Home Assistant via MQTT discovery - no YAML. The password is stored on the Pi and never shown again; leave it blank to keep the current one.

Notifications (ntfy)

Phone alerts away from home via the free ntfy app: pick a hard-to-guess topic, subscribe to it in the app, and Save. Debounce ignores momentary spikes; repeat re-notifies while an alarm is active; offline alert fires if the controller stops reporting (0 = off).
